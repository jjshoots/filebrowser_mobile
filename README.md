# File Browser — Android client (Flutter)

A self-hosted [File Browser](https://github.com/filebrowser/filebrowser) mobile
client with:

- **Biometric login** — credentials are stored in the Android Keystore /
  EncryptedSharedPreferences and unlocked with fingerprint/face (device PIN
  fallback). On unlock the app mints a fresh JWT via `/api/login`.
- **Browse** directories served by File Browser.
- **Background upload/download** that keeps running when the app is backgrounded
  or closed, via a native foreground service with progress notifications.

> Status: v1 scaffold. The Dart application code is complete; the native Android
> project scaffolding is generated by `flutter create` (see Setup below) because
> the Flutter toolchain was not available when this was authored.

## How it talks to the server

| Action     | Request                                                              |
| ---------- | ------------------------------------------------------------------- |
| Login      | `POST /api/login` JSON `{username,password,recaptcha}` → JWT (text)  |
| Renew      | `POST /api/renew` with `X-Auth` (when server sends `X-Renew-Token`)  |
| List dir   | `GET /api/resources/<path>` → JSON listing                          |
| Download   | `GET /api/raw/<path>` (`?algo=zip` to bundle a directory)           |
| Upload     | `POST /api/resources/<path>?override=true`, raw bytes as the body   |

All authenticated requests send the JWT in the `X-Auth` header.

## Project layout

```

  main.dart
  src/
    app.dart                     # MaterialApp + auth gate (setup/lock/browser)

      models.dart                # FbResource, FbUser

    ui/
      login_screen.dart          # first-run setup + lock screen
      browser_screen.dart        # directory browser, download, upload
```

## Setup (toolchain not yet installed on this machine)

1. **Install the toolchain**

   ```bash
   # JDK 17 (required by recent Android Gradle Plugin)
   sudo apt-get install -y openjdk-17-jdk

   # Flutter SDK
   git clone https://github.com/flutter/flutter.git -b stable ~/flutter
   export PATH="$HOME/flutter/bin:$PATH"   # add to ~/.config/fish/config.fish

   # Android command-line tools + SDK, then accept licenses
   flutter doctor            # tells you exactly what's missing
   flutter doctor --android-licenses
   ```

2. **Generate the native Android project** (fills in `android/`, keeps `lib/`
   and `pubspec.yaml`):

   ```bash
   cd /home/jet/Sandboxes/filebrowser_mobile
   flutter create . --platforms=android --org com.eluve.filebrowser
   flutter pub get
   ```

3. **Apply the required Android config** — see the next section, then:

   ```bash
   flutter run            # on a connected device/emulator
   flutter build apk      # release APK in build/app/outputs/flutter-apk/
   ```

## Required Android configuration

After `flutter create`, edit `android/app/src/main/AndroidManifest.xml`.

Add these permissions above `<application>`:

```xml
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.USE_BIOMETRIC"/>
<uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>
<!-- background_downloader foreground service -->
<uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_DATA_SYNC"/>
```

`local_auth` requires the host Activity to be a `FlutterFragmentActivity`. Edit
`android/app/src/main/kotlin/.../MainActivity.kt`:

```kotlin
import io.flutter.embedding.android.FlutterFragmentActivity
class MainActivity : FlutterFragmentActivity()
```

Set `minSdkVersion` to at least **23** (biometrics) in
`android/app/build.gradle` (or `build.gradle.kts`).

If your File Browser instance is served over plain HTTP (no TLS), Android blocks
cleartext by default — either use HTTPS (recommended) or add a network-security
config and `android:usesCleartextTraffic="true"`.

## Notes / next steps

- `background_downloader`'s binary upload is selected via `post: 'binary'`;
  confirm the parameter name against the installed package version (it is stable
  in 8.x).
- Downloads land in the app's documents directory. A "save to / share" flow and
  an explicit transfers screen (listening to `TransferService.updates`) are good
  v1.1 additions.
- Token renewal is handled lazily before requests; for very long-running uploads
  consider refreshing the `X-Auth` header per task.
